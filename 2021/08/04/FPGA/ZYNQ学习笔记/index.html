ZYNQ学习笔记

发表于 2021-08-04 更新于 2021-08-27 阅读次数： Valine：

ZYNQ开发板简介

所用开发板型号：xc7z020clg400-2

PS: 处理系统 （Processing System) , 就是与 FPGA 无关的 ARM 的 SOC 的部分。
PL: 可编程逻辑 (Progarmmable Logic), 就是 FPGA 部分。

ZYNQ的开发是先硬件后软件的方法。具体流程如下：
1) 在 Vivado 上新建 工程，增加一个嵌入式的源文件。
2) 在 Vivado 里添加和配置 PS 和 PL 部分基本的外设，或需要添加自定义的外设。
3) 在 Vivado 里生成顶层 HDL 文件，并添加约束文件。再编译生成比特流文件（*.bit ）。
4) 导出 硬件信息 到 SDK 软件开发环境，在 SDK 环境里可以编写一些调试软件验证硬件
和软件，结合比特流文件单独调试 ZYNQ 系统。
5) 在 SDK 里生成 FSBL 文件。
6) 在 VMware 虚拟机里生成 u boot.elf 、 bootloader 镜像。
7) 在 SDK 里通过 FSBL 文件 , 比特流文件 system.bit 和 u boot.elf 文件生成一个 BOOT .bin
文件。
8) 在 VMware 里生成 Ubuntu 的内核镜像文件 Zimage 和 Ubuntu 的 根 文件系统。另外还
需要要对 FPGA 自定义的 IP 编写驱动。
9) 把 BOOT 、内核、设备树、根文件系统 文件放入到 SD 卡 中，启动开发板电源， Linux
操作系统会从 SD 卡里启动。

引脚约束xdc文件编写

1.将led1信号放在芯片的W13引脚

set_property PACKAGE_PIN W13 [get_port led1]

2.将led1所在引脚的电气标准设置为LVCMOS33：

set_property IOSTANDARD LVCMOS33 [get_ports led1]

锁相环理解

将晶振信号（假设频率为f）与压控振荡器产生的频率信号（频率为Nf）的N分频信号（频率为f）进行相位比较，根据相位的先后来调整压控振荡器的频率，最终当相位相同时达到稳定，即锁相。

RAM和ROM

ram相当于运行内存，即断电后数据丢失。rom相当于硬盘，断电后数据不丢失。

VITIS(新版SDK)使用步骤

我的软件版本为vivado 2020.2，vitis 2020.2 ,vitis HLS 2020.2

PL端：

  1. 在block design 中完成IP配置及连线、

  2. 在suorce界面右键block设计，选择Create HDL Wrapper…，生成顶层hdl文件

  3. 再右键block设计，选择 Generate Output Products，生成block的输出文件，包括IP ，例化模板 RTL 源文件， XDC 约束，第三方综合源文件等等。

  4. generate bitstream

  5. 在菜单栏“ File —> Export —> Export Hardware…”导出硬件信息，这里就包含了 PS 端的配置信息。（若需要PL端参与，则选择include bitstream）。

    （老版vivado此时会生成sdk文件，其中的 .hdf 文件就包含了vivado硬件设计的信息，供软件开发人员使用。新版是生成 .xsa 文件）

PS端：

  1. 直接打开vitis，或者从vivado中 Tools –> Lauch Vitis IDE 启动

  2. 在vitis中 File –> New Platform Project, 导入之前生成的 .xsa文件。生成的platform中包含了硬件信息和软件环境信息

  3. File –> New Application Project，创建应用工程

  4. 在创建完工程后 需要右键platform和application，选择build project。此时platform才会不显示out of date，application才能运行和生成Debug文件夹

  5. 固化：创建Vitis工程的时候，记得勾选Generate boot components，这样可以自动生成FSBL文件。

    ​ 然后右键application，选择 Create Boot Image, 文件顺序为 fsbl.elf –> . bit (有pl则加)–> app.elf

    ​ 最后右键application，选择 Program Flash Memory 即可（记得勾选下面的Blank check after erase 和 Verify after flash）

MIO、EMIO、AXI 总线控制GPIO的不同

MIO和EMIO都是PS端的，MIO直接通过PS操纵PS端的外设（如GPIO），不需要经过PL；EMIO则通过连接到PL端，进而控制PL端的外设。

也可以采用AXI总线的方式进行PS和PL之间的通信。
